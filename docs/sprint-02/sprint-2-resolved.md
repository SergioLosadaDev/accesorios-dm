# 1. Desing Thinking Aplicado.

## Fase 1: Empatizar

**Contexto:** Accesorios D&M actualmente vende sus productos a través de Instagram y whatsapp, los pedidos se gestionan manualmente, lo que genera demoras en respuestas, posibles errores en disponibilidad y dificultad para organizar el inventario y los pedidos confirmados. Actualmente, las ventas se manejan por mensajes uno a uno, lo que hace que el proceso sea desordenado y difícil de controlar.

**MAPA DE EMPATIA:**

**Dice:** "No tengo un sistema donde se registren automáticamente los pedidos."<br>
**Piensa:** "Necesito algo organizado que me permita controlar ventas e inventario."<br>
**Hace:** Anota pedidos manualmente, revisa mensajes uno por uno y confirma disponibilidad manualmente.<br>
**Siente:** Preocupación y estrés por posibles errores y pérdida de ventas.

## Fase 2: Definir

**Problem Statement**<br>
El dueño de Accesorios D&M necesita un sistema centralizado para gestionar pedidos e inventario, porque actualmente el manejo manual mediante mensajes genera desorganización, errores en disponibilidad y dificultad para escalar el negocio.

## Fase 3: Idear

- Catalogo de productos con precio, disponibilidad, imagen y descripción.
- Menu por categorias.
- Carrito de compras editable y persistente.
- Registro de Usuario.
- Proceso de pago.
- Historial de compras.
- Gestion de inventario en tiempo real.
- Estado de pedido
- Integración con whatsapp para SAC.
- Calculo Automatico de Envio.
- CMS para el administrador.

# 2. Priorización MoSCoW

| Prioridad   | Funcionalidad                        | Jusificación                                                    |
| ----------- | ------------------------------------ | --------------------------------------------------------------- |
| Must Have   | Catalogo de productos                | Es la vidriera digital del negocio sin él no hay venta.         |
| Must Have   | Carrito de compras                   | Permite agrupar múltiples accesorios en una sola compra.        |
| Must Have   | Proceso de pago                      | Es el cierre de la venta y la generación de ingresos.           |
| Must Have   | CMS para el administrador            | Permite al dueño actualizar productos sin saber programar.      |
| Should Have | Menu por categorias                  | Ayuda a los clientes a navegar la variedad de productos.        |
| Should Have | Gestion de inventario en tiempo real | Evita la sobreventa de productos con stock limitado.            |
| Should Have | Calculo automatico de envio          | Reduce el abandono del carrito al mostrar el costo final.       |
| Could Have  | Registro de Usuario                  | Ayuda a fidelizar, pero no es necesario para la primera compra. |
| Could Have  | Historial de compras                 | Aporta comodidad al cliente recurrente, no al nuevo.            |
| Won't Have  | Estado de pedido                     | Se puede reemplazar con notificaciones por email/WhatsApp.      |
| Won't Have  | Integración con whatsapp para SAC    | Un simple botón de contacto es suficiente para empezar.         |
